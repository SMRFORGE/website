FAQ

What is SMRFORGE?
SMRFORGE is high-performance simulation software for Small Modular Reactor (SMR) design and analysis. It compresses advanced reactor development timelines by accelerating modeling, prototyping, and Monte Carlo simulation workflows. SMRFORGE provides neutronics, thermal-hydraulics, safety analysis, and burnup in one unified platform.
What's the difference between Community and Pro?
Community is MIT-licensed and includes neutronics, Monte Carlo transport, thermal-hydraulics, safety, burnup, visualization, the CLI, and a one-command React dashboard. Pro adds Serpent/OpenMC/MCNP converters, BenchmarkRunner, ReportGenerator, regulatory traceability, surrogate/ML workflows, and air-gapped deployment.
How does ENDF setup work?
SMRFORGE includes a guided setup wizard and supports bulk storage. You can download ENDF/B-VIII.1 from NNDC or IAEA and point the tool to your directory. The wizard walks you through configuration.
Is Docker or air-gapped deployment supported?
Yes. Community supports Docker. Pro includes air-gapped bundles for regulated environments without internet access.
What Python version is required?
Python 3.11 or higher. Works with standard pip, uv, or conda.
How do I launch the SMRFORGE dashboard?
Install with pip install smrforge[dashboard] and run smrforge serve. The React-based dashboard opens at http://127.0.0.1:8765 in one process — the same FastAPI gateway serves both the API and the SPA. The dashboard ships inside the wheel with no CDN runtime fetches, so it works in fully air-gapped environments.
